# Style Guide

## Naming Conventions
- Use kebab-case for all folder names.
- Use standard filenames when applicable:
  - `prompt_strict.md` for strict execution mode
  - `usage.md` for practical usage notes
  - `README.md` for directory-level overview

## Prompt Structure
Each production prompt should include:
1. Title
2. Objective or mode definition
3. Hard constraints
4. Allowed changes/actions
5. Decision priorities (ordered)
6. Final quality principle

## Language Rules
- All repository content must be written in English.
- Do not mix languages inside repository files.
- Keep wording precise, direct, and operational.
